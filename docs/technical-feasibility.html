System audit15 systems · 4 primaries shown

Each metric in the BOS surface has a designated source-of-truth system. The four below are the foundation; the remaining 11 (Hadreen, Assist App, RFID, T8 Knowledge, Bolt, Arise, Power BI, SF Commerce, MPOS, Excel-Vault stitches, WhatsApp ops) feed in as secondary or read-only inputs.

ARES / BOS

source of truth

Foundation for the Pulse feed. Primary for transactions. Reported peak-hour freezes in the field; manual interpretation required for segment breakdown — which is why SFA011 still runs as a separate PDF in BBW today.

SIOCS

inventory primary

Best truth for inventory. Primary for the unified stock layer. Recommended refresh cadence: 90s polling. SF Commerce stays as fallback only for ecom-exclusive SKUs.

OMS

orders

Required for the Assisted Ecom write-path. Endless Aisle logic depends on the OMS real-time funnel. 2FA login overhead today; needs service-account fix for BOS read.

Reporting Vault

historical · slow

Slow load times. Source for P&L and weekly BP targets. Store managers currently apply mental-math overrides daily because of lag — surfaced in BOS as the Edit Target button rather than an off-screen workaround.

No tech stack changes required for AME onboarding. AME KPI extensions — Aura linkage, RFID manual-exception handling, W22 business-plan reconciliation — are handled through the existing BOS data layer. AME enters the same role-aware surface as the other 7 brands; the only additions are AME-specific KPI extensions on top of the shared stack.
